# UX Audit — 2026-04-20

Full findings from code-based user journey review (no live browser access).

## Issues by Page

### home.blade.php
- ad-unit 'home_top' is included TWICE (line 41 and line 119) — duplicate ad call
- Share code input has min-length check of 4 but codes are 8 chars — inconsistent validation message risk
- $default variable could be null if no default board is seeded — @if($default) guards exist, OK

### games/show.blade.php
- Uses @section('description') instead of @section('meta_description') — layout uses 'meta_description' so this section is never rendered; the game room page will always show the default meta description
- No og:image anywhere in the layout

### play/show.blade.php
- Setup modal "先編輯棋盤格子" button links to boards.edit — requires auth+email verification — guest user will be bounced to login with no explanation
- Gender radio buttons use ♂ ♀ symbols (Unicode characters) — these render as emoji on some systems

### auth/register.blade.php
- No terms-of-service checkbox before submitting — user registers without explicitly agreeing to terms
- Only shows first validation error ($errors->first()) — user won't see all errors at once

### auth/login.blade.php
- "忘記密碼？" link exists (good)
- Rate limit: 5 attempts per 60s, then shows countdown in seconds (good UX)

### auth/verify-email.blade.php
- No link back to home or to continue browsing as guest — user is stuck on this page until they verify or log out

### legal/privacy.blade.php
- Contains literal placeholder text: `[請填入聯絡信箱]` — not replaced before deployment

### legal/terms.blade.php
- Contains literal placeholder text: `[請填入聯絡信箱]` — not replaced before deployment

### public/robots.txt
- Sitemap line hardcoded: `Sitemap: https://yourdomain.com/sitemap.xml` — must be updated before launch

### layouts/app.blade.php
- No og:image meta tag anywhere — sharing to social media will show no image
- nav "開始遊戲" link goes to /play (custom board), not /games — could confuse users expecting Flying Chess
- Age gate uses display:none initially then JS shows it — could cause layout flash on slow connections

### sitemap/index.blade.php
- Includes /games (hourly changefreq) — game lobby is dynamic and has no SEO value; may waste crawl budget
- Does not include /privacy or /terms canonical URLs with noindex consideration
